## `omni::module`

`omni::module` — lightweight view одного загруженного image из process loader list.

| Method | Purpose |
| --- | --- |
| `entry()`, `loader_entry()` | доступ к underlying loader entry |
| `image()` | возвращает `win::image*` для доступа к PE headers |
| `base_address()`, `native_handle()` | возвращают base address модуля |
| `entry_point()` | возвращает image entry point |
| `name()`, `wname()` | возвращают ANSI или wide module names |
| `system_path()` | возвращает полный путь к модулю |
| `exports()` | возвращает `omni::module_exports` |
| `present()`, `operator bool()` | проверяют валидность module view |
| `matches_name_hash(hash)` | сравнивает с hashed module name |

Для форматирования доступны `std::formatter<omni::module>` и stream output.

## `omni::modules`

`omni::modules` — bidirectional range поверх PEB loader list.

| Method | Purpose |
| --- | --- |
| `begin()`, `end()` | итерация по загруженным модулям |
| `skip(count = 1)` | пропустить начальные entries loader list |
| `find(module_name_hash)` | поиск по hash имени модуля |
| `find(omni::address)` | поиск по base address |
| `find_if(predicate)` | поиск по custom predicate |
| `contains(...)` | проверка, загружен ли модуль |

## Convenience functions

- `omni::base_module()` возвращает module текущего process image.
- `omni::get_module(module_name_hash)` ищет загруженный module по имени.
- `omni::get_module(omni::address)` ищет module по base address.

Name matching принимает и `kernel32.dll`, и `kernel32`, если фактическое имя модуля имеет суффикс `.dll`.

## Пример

```cpp
#include "omni/modules.hpp"

#include <print>
#include <ranges>

int main() {
  auto self = omni::base_module();
  auto kernel32 = omni::get_module(L"kernel32.dll");

  std::println("current image: {}", self.name());
  std::println("kernel32 base: {:#x}", kernel32.base_address().value());

  for (const auto& module : omni::modules{} | std::views::take(5)) {
    std::println("{}", module.name());
  }
}
```
