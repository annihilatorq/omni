## Подключение

`omni` — header-only библиотека. Достаточно добавить директорию проекта в CMake и подключить target `omni::omni`.

```cmake
add_subdirectory(path/to/omni)
target_link_libraries(your_target PRIVATE omni::omni)
```

## Требования

- Windows
- CMake 3.21+
- C++23 compiler

## Минимальный пример

```cpp
#include <Windows.h>

#include "omni/modules.hpp"
#include "omni/syscall.hpp"

#include <cstdint>
#include <print>

int main() {
  auto kernel32 = omni::get_module(L"kernel32.dll");
  auto yield_status = omni::syscall("NtYieldExecution");

  std::println("module  : {}", kernel32.name());
  std::println("exports : {}", kernel32.exports().size());
  std::println("yield   : 0x{:08X}", static_cast<std::uint32_t>(yield_status.value));
}
```

## Области API

| Раздел | Основные сущности |
| --- | --- |
| Utilities | `omni::address`, `omni::fnv1a32`, `omni::fnv1a64`, `omni::default_hash`, `omni::error` |
| Status | `omni::status`, `omni::severity`, `omni::facility`, `omni::ntstatus` |
| Modules | `omni::module`, `omni::modules`, `omni::get_module` |
| Exports | `omni::module_export`, `omni::module_exports`, `omni::get_export`, `omni::try_get_export` |
| Lazy Import | `omni::lazy_importer`, `omni::lazy_import` |
| Syscalls | `omni::syscaller`, `omni::syscall` |
| Shared Data | `omni::shared_user_data`, `omni::win::kernel_user_shared_data` |
| Memory | `omni::nt_allocator`, `omni::rw_allocator`, `omni::rx_allocator`, `omni::rwx_allocator` |

## Конфигурация

Поддерживаются compile-time switches:

| Макрос | Эффект |
| --- | --- |
| `OMNI_DISABLE_CACHING` | отключает внутренний cache для lazy import и syscall IDs |
| `OMNI_ENABLE_ERROR_STRINGS` | сохраняет строковые описания ошибок в non-debug сборках |
| `OMNI_DISABLE_ERROR_STRINGS` | убирает строковые описания ошибок даже в debug |

## Расширяемость

Публичный API `omni` не зафиксирован только на одном стиле использования. Основные точки расширения:

- custom hash type вместо `omni::default_hash`;
- typed и compile-time overloads у `lazy_import` и `syscall`;
- настройка allocator policy через `AllocFlags` и `Protect`;
- compile-time feature toggles через макросы.

Если нужен единый взгляд на эти возможности, смотри раздел `Utilities`, где отдельно описана замена hash policy и связанные точки кастомизации.

## Ограничения

- библиотека ориентирована на Windows;
- `syscall` API рассчитан на x64;
- строковые литералы в hash-aware overloads могут конвертироваться в hash типы на compile time.
