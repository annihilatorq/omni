## Ограничение платформы

`omni::syscaller` и `omni::syscall` доступны только на x64.

## `omni::syscaller<T>`

Typed wrapper, который:

1. находит экспорт в loaded modules;
2. извлекает syscall ID из syscall stub;
3. подготавливает shellcode-bridge;
4. выполняет системный вызов.

### Конструктор

- `syscaller(export_name_hash)`

### Основные методы

| Метод | Возвращает | Назначение |
| --- | --- | --- |
| `try_invoke(args...)` | `std::expected<T, std::error_code>` | безопасный вызов с диагностикой |
| `invoke(args...)` | `T` | вызов без явной обработки ошибки |
| `operator()(args...)` | `T` | shorthand вызов |

## `omni::syscall`

Свободные overloads для краткой записи.

### Overloads

- `omni::syscall<T>(export_name_hash, args...)`
- `omni::syscall<F>(export_name_hash, args...)`
- `omni::syscall<Func, Hasher>(args...)`
- `omni::syscall<Func>(args...)`

## Примеры

### Generic status

```cpp
auto status = omni::syscall("NtYieldExecution");
```

### Typed syscall wrapper

```cpp
using nt_query_info_process_fn = omni::status (*)(HANDLE, ULONG, void*, ULONG, ULONG*);
omni::syscaller<nt_query_info_process_fn> query_process{"NtQueryInformationProcess"};
```

### Typed free-function overload

```cpp
auto status = omni::syscall<nt_query_info_process_fn>(
  "NtQueryInformationProcess",
  ::GetCurrentProcess(),
  0U,
  &process_info,
  sizeof(process_info),
  &return_length
);
```

## Ошибки

В `try_invoke()` возможны:

- `omni::error::export_not_found`
- `omni::error::module_not_loaded`
- `omni::error::syscall_id_not_found`

`syscall_id_not_found` означает, что экспорт найден, но не распознан как подходящий syscall stub.

## Важные замечания

- поиск syscall ID основан на паттерне байтов внутри export stub;
- код не рассматривает сценарии с hook/EDR, меняющими stub;
- свободные overloads без `try_*` возвращают default-constructed значение при неудаче.
