## `omni::use_ordinal_t`

Маркерный тип для overloads поиска по ordinal.

```cpp
auto exp = omni::get_export(1, L"ntdll.dll", omni::use_ordinal);
```

## `omni::forwarder_string`

Представление forwarded export в виде пары:

- `module`
- `function`

### Методы

| Метод | Назначение |
| --- | --- |
| `parse(string_view)` | разобрать строку вида `module.function` |
| `is_ordinal()` | проверить, что forward указывает на ordinal |
| `to_ordinal()` | извлечь ordinal |
| `present()` | проверить, что обе части непустые |

## `omni::module_export`

Описание одного экспорта.

### Поля

- `name`
- `address`
- `ordinal`
- `is_forwarded`
- `forwarder_string`
- `module_base`

### Методы

- `is_ordinal_only()`
- `present()`
- `operator bool()`

## `omni::module_exports`

API для export table конкретного модуля.

### Основные методы

| Метод | Назначение |
| --- | --- |
| `size()` | число экспортируемых функций |
| `directory()` | указатель на export directory |
| `name(index)` | имя экспорта по function index |
| `ordinal(index)` | ordinal по index |
| `address(index)` | адрес экспорта |
| `is_export_forwarded(address)` | проверить forwarded export |
| `all()` | range по всем экспортам |
| `ordinal()` | range по ordinal exports |
| `named()` | range по named exports |
| `find(export_name_hash)` | найти по hashed name |
| `find(ordinal, omni::use_ordinal)` | найти по ordinal |
| `find_if(predicate)` | найти по предикату |

### Range-итерация

`omni::module_exports` и его под-range можно использовать как обычные C++ ranges.

```cpp
auto kernel32 = omni::get_module(L"kernel32.dll");

for (const auto& export_entry : kernel32.exports().named()) {
  std::println("{}", export_entry.name);
}
```

## `omni::get_export`

Возвращает `omni::module_export` или пустой результат.

### Overloads

- `omni::get_export(export_name_hash, omni::module module)`
- `omni::get_export(export_name_hash, module_name_hash)`
- `omni::get_export(export_name_hash)`
- `omni::get_export(ordinal, omni::module module, omni::use_ordinal)`
- `omni::get_export(ordinal, module_name_hash, omni::use_ordinal)`

### Особенности

- если экспорт forwarded, библиотека пытается разрешить конечную цель автоматически;
- если целевой модуль forwarder-а не загружен, результат может остаться forwarded export без конечного адреса.

## `omni::try_get_export`

То же, что `get_export`, но с диагностикой через `std::expected<module_export, std::error_code>`.

### Возможные ошибки

- `omni::error::module_not_loaded`
- `omni::error::export_not_found`
- `omni::error::forwarder_string_invalid`

### Пример

```cpp
auto result = omni::try_get_export("GetCurrentProcessId", L"kernel32.dll");
if (!result) {
  std::println("{}", result.error().message());
}
```
