## Ordinal lookup

`omni::use_ordinal_t` и `omni::use_ordinal` выбирают overloads для поиска по ordinal:

```cpp
auto exp = omni::get_export(1, L"ntdll.dll", omni::use_ordinal);
```

## `omni::forwarder_string`

Представляет forwarded export string как части `module` и `function`.

| Method | Purpose |
| --- | --- |
| `parse(string_view)` | разобрать `module.function` |
| `is_ordinal()` | проверить, указывает ли target на ordinal |
| `to_ordinal()` | преобразовать target в ordinal |
| `present()` | проверить, что обе части не пустые |

## `omni::module_export`

| Field | Meaning |
| --- | --- |
| `name` | имя export, пустое для ordinal-only exports |
| `address` | resolved address или raw forwarded address |
| `ordinal` | export ordinal |
| `is_forwarded` | является ли export forwarder |
| `forwarder_string` | parsed forwarder target |
| `forwarder_api_set` | API-set contract data для API-set forwarders |
| `module_base` | base address owning module |

Methods: `is_ordinal_only()`, `present()` и `operator bool()`. Для форматирования доступен `std::formatter<omni::module_export>`.

## `omni::module_exports`

| Method | Purpose |
| --- | --- |
| `size()`, `directory()` | metadata export table |
| `name(index)`, `ordinal(index)`, `address(index)` | чтение raw entries по index |
| `is_export_forwarded(address)` | проверить, указывает ли address на forwarder string |
| `all()` | range по всем exports |
| `ordinal()` | range по ordinal slots |
| `named()` | range по named exports |
| `find(export_name_hash)` | поиск по name/hash |
| `find(ordinal, omni::use_ordinal)` | поиск по ordinal |
| `find_if(predicate)` | поиск по predicate |

## Free lookup

`omni::get_export` возвращает заполненный `omni::module_export` или empty value.

- `omni::get_export(export_name_hash, omni::module module)`
- `omni::get_export(export_name_hash, module_name_hash)`
- `omni::get_export(export_name_hash)`
- `omni::get_export(ordinal, omni::module module, omni::use_ordinal)`
- `omni::get_export(ordinal, module_name_hash, omni::use_ordinal)`

Forwarders разбираются в явные fields. Если target module загружен, lookup может вернуть final target; иначе export остается readable forwarder data.

## Пример

```cpp
auto kernel32 = omni::get_module(L"kernel32.dll");

for (const auto& export_entry : kernel32.exports().named() | std::views::take(5)) {
  std::println("{}", export_entry.name);
}

auto by_name = kernel32.exports().find("GetCurrentProcessId");
auto by_ordinal = kernel32.exports().find(1, omni::use_ordinal);
```
