## `omni::address`

`omni::address` is a small wrapper over a memory address.

| API | Purpose |
| --- | --- |
| constructors | from `nullptr`, `std::uintptr_t`, pointer, or contiguous range |
| `ptr<T>(offset = 0)` | return a typed pointer at an offset |
| `value()` | return the raw `std::uintptr_t` |
| `offset<T>(delta = 0)` | produce an offset address or pointer |
| `as<T>()` | cast the address to another type |
| `span<T>(count)` | view memory as `std::span` |
| `is_in_range(start, end)` | check whether the address is inside a range |
| `invoke<T>(args...)` | call the address as a function |

It supports explicit boolean conversion, comparisons, arithmetic and bitwise operators, stream output, and `std::formatter<omni::address>`.

```cpp
omni::address base{::GetModuleHandleW(nullptr)};
auto* image = base.ptr<void>();
auto entry = base.offset(0x1000);
```

## Hash API

### Types and functions

- `omni::fnv1a32`
- `omni::fnv1a64`
- `omni::default_hash`, an alias for `omni::fnv1a64`
- `omni::hash_pair<Hasher = default_hash>`
- `omni::hash<Hasher>(value)`

Hashes are ASCII case-insensitive, which makes them suitable for Windows module and export names.

```cpp
constexpr omni::fnv1a64 kernel32{"kernel32.dll"};
auto export_hash = omni::hash<omni::fnv1a32>("GetCurrentProcessId");
constexpr omni::hash_pair<> lookup{"GetSystemDirectoryW", "kernel32.dll"};
```

### Literals

The `omni::literals` namespace provides string literal operators for narrow and wide strings:

- `"..."_fnv1a32`
- `"..."_fnv1a64`
- `"..."_hash`

### Formatting

`std::formatter` support is available for `omni::fnv1a32` and `omni::fnv1a64`.

## Custom hash policies

Most hash-aware APIs are templated on `Hasher`. The same policy can be used with modules, exports, lazy imports, and syscalls.

A hasher must satisfy `omni::concepts::hash`: it needs a `value_type`, `value()`, comparisons, construction from strings/values, and hashing for supported string or span inputs.

```cpp
using my_hash = omni::fnv1a32;

auto module = omni::get_module(my_hash{L"kernel32.dll"});
auto export_entry = omni::get_export(my_hash{"GetCurrentProcessId"}, my_hash{"kernel32.dll"});
auto result = omni::lazy_import<void, my_hash>(my_hash{"SetLastError"}, 0U);
```

### Implementing your own hasher

A custom hasher is a small value type, not just a free function. It stores a computed value and can also be called as a runtime hashing functor.

Minimal contract:

- define `using value_type = ...`;
- provide a default constructor;
- construct from `value_type`;
- construct from narrow and wide compile-time string literals;
- construct from hashable runtime views such as `std::string_view`;
- implement `operator()(omni::concepts::hashable)`;
- expose `value()`;
- support comparison with another hasher and with `value_type`.

```cpp
template <std::unsigned_integral T>
class djb2_hash {
  constexpr static auto initial_value = T{5381};

 public:
  using value_type = T;

  constexpr djb2_hash() = default;
  constexpr explicit(false) djb2_hash(value_type value): value_(value) {}

  template <typename CharT, std::size_t N>
  consteval explicit(false) djb2_hash(const CharT (&string)[N]) {
    for (std::size_t i{}; i < N - 1; i++) {
      value_ = append(value_, string[i]);
    }
  }

  consteval explicit(false) djb2_hash(omni::concepts::hashable auto string) {
    for (std::size_t i{}; i < string.size(); i++) {
      value_ = append(value_, string[i]);
    }
  }

  [[nodiscard]] value_type operator()(omni::concepts::hashable auto object) {
    auto value = initial_value;
    for (std::size_t i{}; i < object.size(); i++) {
      value = append(value, object[i]);
    }
    return value;
  }

  [[nodiscard]] value_type value() const {
    return value_;
  }

  [[nodiscard]] constexpr bool operator==(value_type other) const {
    return value_ == other;
  }

  [[nodiscard]] constexpr bool operator==(const djb2_hash& other) const {
    return value_ == other.value_;
  }

 private:
  template <typename CharT>
  [[nodiscard]] constexpr static value_type append(value_type accumulator, CharT character) {
    return (accumulator * 33) + static_cast<value_type>(to_lower(character));
  }

  template <typename CharT>
  [[nodiscard]] constexpr static CharT to_lower(CharT character) {
    return ((character >= 'A' && character <= 'Z') ? (character + 32) : character);
  }

  value_type value_{initial_value};
};

using djb2_64 = djb2_hash<std::uint64_t>;
static_assert(omni::concepts::hash<djb2_64>);
```

### Passing a custom hasher to omni APIs

Once the type satisfies `omni::concepts::hash`, pass it either as the hash value type or as the explicit `Hasher` template parameter.

```cpp
using my_hash = djb2_64;

auto kernel32 = omni::get_module(my_hash{L"kernel32.dll"});
auto export_entry = omni::get_export(my_hash{"GetCurrentProcessId"}, my_hash{"kernel32.dll"});

using get_current_process_id_fn = DWORD(WINAPI*)();
auto process_id = omni::lazy_import<get_current_process_id_fn, my_hash>(my_hash{"GetCurrentProcessId"});

auto status = omni::syscall<omni::status, my_hash>(my_hash{"NtYieldExecution"});
constexpr omni::hash_pair<my_hash> pair{"GetModuleHandleW", "kernel32.dll"};
```

Use the same hasher everywhere in a lookup chain. For example, do not build a `hash_pair<my_hash>` and then compare it against `omni::default_hash` values.

## Concepts

Public concepts live in `omni/concepts.hpp`:

- `omni::concepts::hash`
- `omni::concepts::hashable`
- `omni::concepts::function_pointer`

## Error API

`enum class omni::error` values:

- `module_not_loaded`
- `export_not_found`
- `syscall_id_not_found`
- `forwarder_string_invalid`

The errors integrate with `std::error_code` through `omni::error_category()`, `omni::make_error_code()`, and the standard `std::is_error_code_enum` specialization.

```cpp
std::error_code ec = omni::error::export_not_found;
```
