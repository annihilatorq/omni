## Platform limitation

`omni::syscaller` and `omni::syscall` are available only on x64.

## `omni::syscaller<T>`

A typed wrapper that:

1. finds the export in loaded modules;
2. extracts the syscall ID from the syscall stub;
3. prepares a shellcode bridge;
4. performs the system call.

### Constructor

- `syscaller(export_name_hash)`

### Main methods

| Method | Returns | Purpose |
| --- | --- | --- |
| `try_invoke(args...)` | `std::expected<T, std::error_code>` | safe call with diagnostics |
| `invoke(args...)` | `T` | call without explicit error handling |
| `operator()(args...)` | `T` | shorthand invocation |

## `omni::syscall`

Free overloads for concise usage.

### Overloads

- `omni::syscall<T>(export_name_hash, args...)`
- `omni::syscall<F>(export_name_hash, args...)`
- `omni::syscall<Func, Hasher>(args...)`
- `omni::syscall<Func>(args...)`

## Examples

### Generic status

```cpp
auto status = omni::syscall("NtYieldExecution");
```

### Typed syscall wrapper

```cpp
using nt_query_info_process_fn = omni::status (*)(HANDLE, ULONG, void*, ULONG, ULONG*);
omni::syscaller<nt_query_info_process_fn> query_process{"NtQueryInformationProcess"};
```

### Typed free-function overload

```cpp
auto status = omni::syscall<nt_query_info_process_fn>(
  "NtQueryInformationProcess",
  ::GetCurrentProcess(),
  0U,
  &process_info,
  sizeof(process_info),
  &return_length
);
```

## Errors

In `try_invoke()`, the following are possible:

- `omni::error::export_not_found`
- `omni::error::module_not_loaded`
- `omni::error::syscall_id_not_found`

`syscall_id_not_found` means the export was found, but was not recognized as a suitable syscall stub.

## Important notes

- syscall ID lookup is based on a byte pattern inside the export stub;
- the code does not handle scenarios where hooks/EDR modify the stub;
- free overloads without `try_*` return a default-constructed value on failure.
