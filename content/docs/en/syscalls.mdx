## Platform limitation

`omni::syscaller` and `omni::syscall` are x64-only.

## `omni::syscaller<T>`

A typed wrapper that finds an export, extracts the syscall ID from its stub, prepares a shellcode bridge, and performs the system call.

### Constructor

- `syscaller(export_name_hash, syscall_id_parser parser = default_syscall_id_parser)`
- `syscaller(default_hash, syscall_id_parser parser = default_syscall_id_parser)`

`syscaller<T>::syscall_id_parser` is the parser type. The default parser is `omni::default_syscall_id_parser(const omni::module_export&)`.

### Methods

| Method | Returns | Purpose |
| --- | --- | --- |
| `try_invoke(args...)` | `std::expected<T, std::error_code>` | invoke with diagnostics |
| `invoke(args...)` | `T` | invoke and return default value on failure |
| `operator()(args...)` | `T` | shorthand for `invoke` |

Function-pointer specializations infer arguments and return type from the NT function signature.

## `omni::syscall`

Free overloads provide concise one-off calls:

- `omni::syscall<T = omni::status>(export_name_hash, args...)`
- `omni::syscall<F>(export_name_hash, args...)`
- `omni::syscall<Func, Hasher>(args...)`
- `omni::syscall<Func>(args...)`

## Examples

```cpp
auto yield_status = omni::syscall<omni::status>("NtYieldExecution");
```

```cpp
using nt_query_info_process_fn = omni::status (*)(HANDLE, ULONG, void*, ULONG, ULONG*);

omni::syscaller<nt_query_info_process_fn> query_process{"NtQueryInformationProcess"};
auto result = query_process.try_invoke(
  ::GetCurrentProcess(),
  0U,
  &process_info,
  sizeof(process_info),
  &return_length
);
```

```cpp
auto status = omni::syscall<nt_query_info_process_fn>(
  "NtQueryInformationProcess",
  ::GetCurrentProcess(),
  0U,
  &process_info,
  sizeof(process_info),
  &return_length
);
```

## Errors and notes

`try_invoke()` can report `export_not_found`, `module_not_loaded`, or `syscall_id_not_found`. The last error means the export was found but did not match the expected syscall stub pattern.

Free overloads without `try_*` return a default-constructed value on failure. The default parser does not handle hooks or EDR-modified stubs.
