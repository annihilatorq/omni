## Integration

`omni` is header-only. Add the project directory in CMake and link the exported target:

```cmake
add_subdirectory(path/to/omni)
target_link_libraries(your_target PRIVATE omni::omni)
```

## Requirements

- Windows
- CMake 3.21+
- C++23 compiler

## Minimal example

```cpp
#include <Windows.h>

#include "omni/modules.hpp"
#include "omni/syscall.hpp"

#include <cstdint>
#include <print>

int main() {
  auto kernel32 = omni::get_module(L"kernel32.dll");
  auto yield_status = omni::syscall("NtYieldExecution");

  std::println("module  : {}", kernel32.name());
  std::println("exports : {}", kernel32.exports().size());
  std::println("yield   : 0x{:08X}", static_cast<std::uint32_t>(yield_status.value));
}
```

## API areas

| Area | Main entities |
| --- | --- |
| Utilities | `omni::address`, `omni::fnv1a32`, `omni::fnv1a64`, `omni::default_hash`, `omni::error` |
| Status | `omni::status`, `omni::severity`, `omni::facility`, `omni::ntstatus` |
| Modules | `omni::module`, `omni::modules`, `omni::base_module`, `omni::get_module` |
| Exports | `omni::module_export`, `omni::module_exports`, `omni::get_export` |
| API Sets | `omni::api_set`, `omni::api_sets`, `omni::api_set_host`, `omni::get_api_set` |
| Lazy Import | `omni::lazy_importer`, `omni::lazy_import` |
| Syscalls | `omni::syscaller`, `omni::syscall`, `omni::default_syscall_id_parser` |
| Shared Data | `omni::shared_user_data`, `omni::win::kernel_user_shared_data` |
| Memory | `omni::nt_allocator`, `omni::rw_allocator`, `omni::rx_allocator`, `omni::rwx_allocator` |

## Build examples and tests

```bash
cmake -S . -B build -DOMNI_BUILD_EXAMPLES=ON -DOMNI_BUILD_TESTS=ON
cmake --build build --config Release
ctest --test-dir build --output-on-failure -C Release
```

## Configuration macros

| Macro | Effect |
| --- | --- |
| `OMNI_DISABLE_CACHING` | disables the internal lazy-import and syscall-ID caches |
| `OMNI_ENABLE_ERROR_STRINGS` | keeps readable error strings in non-debug builds |
| `OMNI_DISABLE_ERROR_STRINGS` | strips error strings even in debug builds |

## Notes

- the library is Windows-specific;
- the syscall API is intended for x64 processes;
- hash-aware overloads accept strongly typed hash values and string literals that can be converted at compile time.
