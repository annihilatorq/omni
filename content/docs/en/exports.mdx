## Ordinal lookup

`omni::use_ordinal_t` and `omni::use_ordinal` select ordinal-based overloads:

```cpp
auto exp = omni::get_export(1, L"ntdll.dll", omni::use_ordinal);
```

## `omni::forwarder_string`

Represents a forwarded export string as `module` and `function` parts.

| Method | Purpose |
| --- | --- |
| `parse(string_view)` | parse `module.function` |
| `is_ordinal()` | check whether the target is an ordinal |
| `to_ordinal()` | convert the target to an ordinal |
| `present()` | check that both parts are non-empty |

## `omni::module_export`

| Field | Meaning |
| --- | --- |
| `name` | export name, empty for ordinal-only exports |
| `address` | resolved address or raw forwarded address |
| `ordinal` | export ordinal |
| `is_forwarded` | whether the export is a forwarder |
| `forwarder_string` | parsed forwarder target |
| `forwarder_api_set` | API-set contract data for API-set forwarders |
| `module_base` | owning module base address |

Methods: `is_ordinal_only()`, `present()`, and `operator bool()`. Formatter support is available through `std::formatter<omni::module_export>`.

## `omni::module_exports`

| Method | Purpose |
| --- | --- |
| `size()`, `directory()` | inspect export-table metadata |
| `name(index)`, `ordinal(index)`, `address(index)` | read raw entries by index |
| `is_export_forwarded(address)` | check whether an address points to a forwarder string |
| `all()` | range over all exports |
| `ordinal()` | range over ordinal slots |
| `named()` | range over named exports |
| `find(export_name_hash)` | find by name/hash |
| `find(ordinal, omni::use_ordinal)` | find by ordinal |
| `find_if(predicate)` | find by predicate |

## Free lookup

`omni::get_export` returns a populated `omni::module_export` or an empty value.

- `omni::get_export(export_name_hash, omni::module module)`
- `omni::get_export(export_name_hash, module_name_hash)`
- `omni::get_export(export_name_hash)`
- `omni::get_export(ordinal, omni::module module, omni::use_ordinal)`
- `omni::get_export(ordinal, module_name_hash, omni::use_ordinal)`

Forwarders are parsed into explicit fields. If the target module is loaded, lookup can resolve the final target; otherwise the returned export remains readable as forwarder data.

## Example

```cpp
auto kernel32 = omni::get_module(L"kernel32.dll");

for (const auto& export_entry : kernel32.exports().named() | std::views::take(5)) {
  std::println("{}", export_entry.name);
}

auto by_name = kernel32.exports().find("GetCurrentProcessId");
auto by_ordinal = kernel32.exports().find(1, omni::use_ordinal);
```
