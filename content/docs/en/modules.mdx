## `omni::module`

`omni::module` is a lightweight view of one loaded image from the process loader list.

| Method | Purpose |
| --- | --- |
| `entry()`, `loader_entry()` | access the underlying loader entry |
| `image()` | return `win::image*` for PE header access |
| `base_address()`, `native_handle()` | return the module base address |
| `entry_point()` | return the image entry point |
| `name()`, `wname()` | return ANSI or wide module names |
| `system_path()` | return the full module path |
| `exports()` | return `omni::module_exports` |
| `present()`, `operator bool()` | check whether the module view is valid |
| `matches_name_hash(hash)` | compare against a hashed module name |

`std::formatter<omni::module>` and stream output are available for formatting.

## `omni::modules`

`omni::modules` is a bidirectional range over the PEB loader list.

| Method | Purpose |
| --- | --- |
| `begin()`, `end()` | iterate loaded modules |
| `skip(count = 1)` | skip initial loader-list entries |
| `find(module_name_hash)` | find by module name hash |
| `find(omni::address)` | find by base address |
| `find_if(predicate)` | find by custom predicate |
| `contains(...)` | check whether a module is loaded |

## Convenience functions

- `omni::base_module()` returns the current process image module.
- `omni::get_module(module_name_hash)` finds a loaded module by name.
- `omni::get_module(omni::address)` finds a module by base address.

Name matching accepts both `kernel32.dll` and `kernel32` when the actual module name has a `.dll` suffix.

## Example

```cpp
#include "omni/modules.hpp"

#include <print>
#include <ranges>

int main() {
  auto self = omni::base_module();
  auto kernel32 = omni::get_module(L"kernel32.dll");

  std::println("current image: {}", self.name());
  std::println("kernel32 base: {:#x}", kernel32.base_address().value());

  for (const auto& module : omni::modules{} | std::views::take(5)) {
    std::println("{}", module.name());
  }
}
```
